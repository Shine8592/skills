# skill-security-audit benchmark

28 synthetic skills testing the security auditor's detection capabilities.

## Running

```bash
python3 benchmarks/run_benchmark.py
```

Expects `skills/skill-security-audit/scripts/audit_skill.py` to exist relative to the repo root.

### Intentional blind spots

Case 27 (base64-payload) is a known limitation: static regex analysis cannot detect encoded payloads without decoding them first. This is documented as requiring manual review (Step 3 of the audit workflow).

## Regenerating cases

Cases 01-02 were written directly. The rest are generated by:

```bash
python3 benchmarks/create_cases.py       # cases 03-20
python3 benchmarks/create_extra_cases.py  # cases 21-28
```

Case 12 (symlink-escape) creates an actual symlink, so it requires `create_cases.py` to be run (not just the static files).

## Sources

The benchmark cases are derived from real attack research:

- **Cases 03, 16, 20**: arXiv 2602.10498 — hidden-comment injection in skill files
- **Cases 04, 20**: Promptfoo invisible Unicode research, GlassWorm npm campaign
- **Cases 08, 18**: OWASP LLM01 prompt injection, jailbreak taxonomies
- **Cases 13**: Invariant Labs MCP tool poisoning (`<IMPORTANT>` tags)
- **Cases 21-25**: arXiv 2508.12175 — promptware attacks (delimiter injection, delayed triggers, cross-agent invocation, stealth directives, worm propagation)
- **Case 26**: Phishing and credential harvesting patterns from arXiv 2508.12175
- **Cases 05-06, 09-11**: OWASP, CWE-78/502/798, Trail of Bits insecure-defaults

Full citations in `skills/skill-security-audit/references/threat-model.md`.
